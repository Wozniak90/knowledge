1. Co jsou E-Mail Templates?

SAP S/4HANA (Cloud i On-Premise) obsahuje nativní funkci E-Mail Templates jako součást Output Managementu.
Šablony umožňují:

  • Definovat HTML nebo Plain Text obsah e-mailu přímo v systému (bez hardcodování v ABAPu).
  • Udržovat e-mailový obsah ve více jazycích.
  • Mapovat CDS view pro dynamické proměnné (datum, jméno zákazníka, číslo dokladu atd.).
  • Propojit šablony s output typy přes BRF+ nebo je volat zcela nezávisle na output managementu z vlastního ABAP programu.
  • Jednoduše upravovat obsah e-mailu bez nutnosti ABAP změny a transportu kódu.

3. Systémový předpoklad – SCOT/SAPconnect

⚠️ Bez funkční konfigurace SAPconnect žádný způsob odesílání e-mailů ze SAP nebude fungovat – ani šablony, ani klasický ABAP.

Transakce SCOT – nastavení SMTP

  1. Jdi do transakce SCOT (SAPconnect Administration).
  2. Zkontroluj, zda existuje node typu SMTP (sekce Internet).
  3. Pokud neexistuje, vytvoř ho: Settings → Internet → SMTP Node:
    - SMTP Mail Host – adresa mail serveru (např. smtp.firma.cz)
    - SMTP Port – typicky 25 nebo 587 (TLS)
    - Set Internet Address – e-mailová adresa, která bude vystupovat jako odesílatel SAP systému
  4. Nastav Send Job: tlačítko Jobs → Start Send Process → doporučeno spouštět každých 1–5 minut.

Transakce Popis
SCOT Administrace SAPconnect, nastavení SMTP node

5. Vytvoření E-Mail Template v SE80

Neexistuje dedikovaná transakce – šablona se vytváří jako repository object v SE80:

  1. Otevři SE80.
  2. Vyber balík (nebo Local Objects $TMP) → pravý klik.
  3. Zvolte Create → More → Email Template.
  4. Zadej název (doporučen prefix Z, např. ZBASIS_PENDING_TIME_SHEETS).
  5. Vyplň popis a potvrď uložení do balíku nebo $TMP.

💡 Existující šablony lze prohlížet a editovat přes program SMTG_WB_START (spusť jako SA38/SE38).

Struktura šablony (záložky)

Záložka Popis
Header Název, popis, přiřazení CDS view
Languages Správa šablony v různých jazycích (EN, DE, CS...)
CDS Fields Výběr konkrétních polí z CDS view pro použití v obsahu
Email Subject Předmět e-mailu – může obsahovat {{placeholder}}
Body HTML Tělo e-mailu jako HTML – může obsahovat {{placeholder}}
Body Plain Text Textová alternativa těla e-mailu

Preview šablony

V SE80 je k dispozici tlačítko Preview – umožňuje zobrazit šablonu s testovacími hodnotami ještě před nasazením programu. Velmi užitečné pro ověření HTML layoutu.

6. CDS View pro dynamické proměnné

Pro dynamické proměnné je nutné vytvořit CDS view, která bude zdrojem dat.

Pravidla pro CDS view

  • Musí mít key pole – podle nich se při volání předává CDS klíč (name/value páry).
  • Nemusí být VDM view – postačí @VDM.viewType: #BASIC nebo bez anotace.
  • Pole CDS view se referencují v šabloně jako {{NazevPole}} – název musí přesně odpovídat aliasu/názvu pole v CDS.
  • CDS view musí být aktivní v systému.

Příklad CDS view pro fakturační doklad

@AbapCatalog.sqlViewName: 'ZRSCDS_INVDATA'
@AbapCatalog.compiler.compareFilter: true
@AccessControl.authorizationCheck: #CHECK
@EndUserText.label: 'Invoice Data Line Item wise'
define view ZRSCDS_INVOICE_DATA
as select from vbrk as zzrs_vbrk
join vbrp as zzrs_vbrp
    on zzrs_vbrk.vbeln = zzrs_vbrp.vbeln
{
  key zzrs_vbrk.vbeln,
  key zzrs_vbrp.posnr,
      zzrs_vbrk.fkart,
      zzrs_vbrk.vbtyp,
      @Semantics.currencyCode: true
      zzrs_vbrk.waerk,
      zzrs_vbrk.vkorg,
      zzrs_vbrk.fkdat,
      @Semantics.amount.currencyCode: 'waerk'
      @DefaultAggregation: #SUM
      zzrs_vbrk.netwr,
      zzrs_vbrk.kunag as kunag,
      zzrs_vbrp.fkimg,
      zzrs_vbrp.vrkme,
      zzrs_vbrp.meins,
      zzrs_vbrp.matnr
}

CDS view se přiřazuje v záložce Header šablony. Pole se pak přiřadí v záložce CDS Fields a referencují se v Body/Subject jako {{matnr}}, {{fkdat}} apod.

7. Volání E-Mail Template z ABAPu (jednoduchý případ)

Použij tuto variantu, když potřebuješ odeslat e-mail s jednoduchou šablonou bez speciálních úprav HTML.

Kompletní vzorový kód

TRY.
    " 1. Instance Email API
    DATA(lo_email_api) = cl_smtg_email_api=>get_instance(
        iv_template_id = p_em_id   " ID šablony, např. 'ZINVOICE_NOTIFICATION'
    ).

    " 2. Instance BCS
    DATA(lo_bcs) = cl_bcs=>create_persistent( ).

    " 3. CDS klíč – name/value páry odpovídající key polím CDS view
    DATA(lt_cds_key) = VALUE cl_smtg_email_api=>ty_t_data_key(
        ( name = 'VBELN' value = p_vbeln )
    ).

    " 4. Render šablony do BCS (vloží subject + body automaticky)
    lo_email_api->render_bcs(
        io_bcs       = lo_bcs
        iv_language  = p_spras     " jazyk, např. sy-langu nebo 'E'
        it_data_key  = lt_cds_key
    ).

    " 5. Odesílatel
    DATA(lo_sender) = cl_sapuser_bcs=>create( sy-uname ).
    lo_bcs->set_sender( i_sender = lo_sender ).

    " 6. Příjemce (internetová adresa)
    DATA(lo_recipient) = cl_cam_address_bcs=>create_internet_address( p_rec ).
    lo_bcs->add_recipient( EXPORTING i_recipient = lo_recipient ).

    " 7. Odeslání
    lo_bcs->send( ).
    COMMIT WORK AND WAIT.

CATCH cx_smtg_email_common INTO DATA(lx_email).
    MESSAGE lx_email->get_text( ) TYPE 'E'.
CATCH cx_send_req_bcs INTO DATA(lx_bcs).
    MESSAGE lx_bcs->get_text( ) TYPE 'E'.
CATCH cx_address_bcs INTO DATA(lx_addr).
    MESSAGE lx_addr->get_text( ) TYPE 'E'.
ENDTRY.

8. Vícejazyčné šablony

E-Mail Templates nativně podporují více jazyků – pro každý jazyk lze definovat odlišný předmět i tělo e-mailu.

Jak nastavit vícejazyčnou šablonu v SE80

  1. V šabloně otevři záložku Languages.
  2. Přidej nový jazyk (tlačítko New Language).
  3. Pro každý jazyk zvlášť vyplň:
    - Email Subject
    - Body HTML
    - Body Plain Text

Jak vybrat jazyk při volání

" Jazyk se předává parametrem iv_language
lo_email_api->render_bcs(
    io_bcs       = lo_bcs
    iv_language  = 'E'   " 'E' = angličtina, 'D' = němčina, 'C' = čeština atd.
    it_data_key  = lt_cds_key
).

💡 Pokud pro požadovaný jazyk šablona neexistuje, SAP automaticky padne zpět na fallback jazyk (EN nebo první dostupný). Proto je vhodné vždy mít anglickou verzi jako zálohu.

Doporučení pro vícejazyčné nasazení

  • Udržuj EN jako základní jazyk – bude použit pokud cílový jazyk chybí.
  • Jazyk příjemce lze načíst z user masteru: cl_abap_syst=>get_language( ) nebo z tabulky USR01.
  • Pro zákazníky SAP: jazyk lze načíst z KNA1-SPRAS.

9. Vkládání tabulky s více záznamy do e-mailu

Použij tuto variantu, když potřebuješ do jednoho e-mailu vložit dynamickou tabulku s proměnným počtem řádků (výpis položek, seznam projektů apod.).

Princip

  1. V HTML šabloně definuj tabulku s hlavičkou a HTML komentářem jako placeholderem:
    ```html

`` 2. V ABAPu dynamicky sestav HTML řádky tabulky jako string (lv_content). 3. Použijrender()místorender_bcs()– vrátí raw HTML string. 4. Nahraď placeholder vygenerovaným HTML (REPLACE ... WITH ... INTO). 5. Sestav multipart zprávu a odešli přesCL_BCS`.

HTML šablona – část s tabulkou (vkládej do Body HTML)

<!-- Dynamická tabulka s projekty -->
<table border="1" cellpadding="3"
       style="border-collapse:collapse; font-size:14px; width:100%">
  <thead>
    <tr style="background-color:#F16523; color:#ffffff;">
      <th style="text-align:left; padding:5px;">EmpID</th>
      <th style="text-align:left; padding:5px;">Employee</th>
      <th style="text-align:left; padding:5px;">Project</th>
      <th style="text-align:left; padding:5px;">Planned Hrs</th>
      <th style="text-align:left; padding:5px;">Approved Hrs</th>
      <th style="text-align:left; padding:5px;">Remaining Hrs</th>
    </tr>
  </thead>
  <tbody>
    <!--///EmpContent-->
  </tbody>
</table>

💡 Vždy edituj HTML nejprve v VS Code – máš zvýraznění syntaxe, validaci a náhled. Do SE80 kopíruj až po otestování.

ABAP kód – sestavení HTML tabulky a odeslání

CONSTANTS: c_template_id TYPE smtg_template_id VALUE 'ZBASIS_PENDING_TIMESHEETS',
           c_english      TYPE spras             VALUE 'E'.

" -----------------------------------------------------------------------
" 1. Sestav unikátní seznam zaměstnanců
" -----------------------------------------------------------------------
DATA(lt_emp) = i_output.
SORT lt_emp BY pernr ASCENDING.
DELETE ADJACENT DUPLICATES FROM lt_emp COMPARING pernr.

" -----------------------------------------------------------------------
" 2. Pro každého zaměstnance sestav HTML řádky všech jeho projektů
" -----------------------------------------------------------------------
DATA lt_timsh TYPE TABLE OF zs_timesh_data.
DATA lv_content TYPE string.

LOOP AT lt_emp INTO DATA(ls_emp).
    CLEAR lv_content.

    LOOP AT i_output INTO DATA(ls_output) WHERE pernr = ls_emp-pernr.
        lv_content = |{ lv_content }<tr>|.
        lv_content = |{ lv_content }<td style="padding:4px">{ ls_output-pernr }</td>|.
        lv_content = |{ lv_content }<td style="padding:4px">{ ls_output-pernr_name }</td>|.
        lv_content = |{ lv_content }<td style="padding:4px">{ ls_output-prj_code } – { ls_output-prj_name }</td>|.
        lv_content = |{ lv_content }<td style="padding:4px; text-align:right">{ ls_output-workd }</td>|.
        lv_content = |{ lv_content }<td style="padding:4px; text-align:right">{ ls_output-cat_approved }</td>|.
        lv_content = |{ lv_content }<td style="padding:4px; text-align:right">{ ls_output-cat_remaining }</td>|.
        lv_content = |{ lv_content }</tr>|.
    ENDLOOP.

    DATA(ls_timsh) = VALUE zs_timesh_data(
        pernr       = ls_emp-pernr
        pernr_name  = ls_emp-pernr_name
        pmemail     = ls_emp-pmemail
        emp_content = lv_content
    ).
    APPEND ls_timsh TO lt_timsh.
ENDLOOP.

" -----------------------------------------------------------------------
" 3. Odešli jeden e-mail pro každého zaměstnance
" -----------------------------------------------------------------------
LOOP AT lt_timsh INTO ls_timsh.

    TRY.
        DATA(oref_email_api) = cl_smtg_email_api=>get_instance(
            iv_template_id = c_template_id
        ).
        DATA(oref_bcs) = cl_bcs=>create_persistent( ).

        " CDS klíč pro načtení dynamických proměnných ({{EmployeeName}}, {{TodaysDate}} atd.)
        DATA(i_cds_key) = VALUE cl_smtg_email_api=>ty_t_data_key(
            ( name = 'EMPLOYEE' value = ls_timsh-pernr )
        ).

        " Získej raw HTML – render() vrátí string, nikoli rovnou do BCS
        oref_email_api->render(
            EXPORTING
                iv_language  = c_english
                it_data_key  = i_cds_key
            IMPORTING
                ev_body_html    = DATA(body_html)
                ev_subject      = DATA(lv_subject_tmpl)
        ).

        " Nahraď placeholder dynamicky vygenerovanými řádky tabulky
        REPLACE '<!--///EmpContent-->' WITH ls_timsh-emp_content INTO body_html.

        " Sestav multipart HTML zprávu
        DATA(body_html_soli) = cl_bcs_convert=>string_to_soli( body_html ).
        DATA(multipart) = NEW cl_gbt_multirelated_service( ).
        multipart->set_main_html(
            EXPORTING
                content     = body_html_soli
                description = 'EmailBody'
        ).

        " Předmět e-mailu (lze použít ze šablony nebo sestavit dynamicky)
        DATA(lv_subject) = |Pending Time Sheets – { ls_timsh-pernr_name }|.

        " Vytvoř e-mailový dokument
        DATA(doc_bcs) = cl_document_bcs=>create_from_multirelated(
            EXPORTING
                i_subject          = lv_subject
                i_multirel_service = multipart
        ).
        oref_bcs->set_document( doc_bcs ).

        " Odesílatel (systémový workflow user)
        DATA(oref_sender) = cl_sapuser_bcs=>create( 'SAP_WFRT' ).
        oref_bcs->set_sender( i_sender = oref_sender ).

        " Příjemce
        DATA(oref_recipient) = cl_cam_address_bcs=>create_internet_address(
            ls_timsh-pmemail
        ).
        oref_bcs->add_recipient( EXPORTING i_recipient = oref_recipient ).

        " Odeslání
        oref_bcs->send( ).
        COMMIT WORK AND WAIT.
        WRITE / |>> OK: { ls_timsh-pernr } – { ls_timsh-pernr_name }|.

    CATCH cx_smtg_email_common INTO DATA(lx_e).
        WRITE / |>> CHYBA (šablona): { lx_e->get_text( ) }|.
    CATCH cx_send_req_bcs INTO DATA(lx_b).
        WRITE / |>> CHYBA (BCS): { lx_b->get_text( ) }|.
    CATCH cx_address_bcs INTO DATA(lx_a).
        WRITE / |>> CHYBA (adresa): { lx_a->get_text( ) }|.
    ENDTRY.

ENDLOOP.

10. E-mail s přílohou (PDF/soubor)

E-Mail Templates lze kombinovat s přílohami. Přílohu je nutné přidat do instance CL_BCS jako binární dokument.

Princip

" Předpokládáme, že lv_pdf_xstring obsahuje PDF jako xstring (např. z cl_rspo_pdf)
DATA(lt_solix)  = cl_bcs_convert=>xstring_to_solix( lv_pdf_xstring ).

DATA(lo_attachment) = cl_document_bcs=>create_document(
    i_type        = 'PDF'
    i_text        = lt_solix
    i_length      = xstrlen( lv_pdf_xstring )
    i_subject     = 'Invoice.pdf'
).

" Přidání přílohy k hlavní zprávě
lo_bcs->set_document( lo_main_doc ).
lo_bcs->set_document( lo_attachment ).   " ← přidá jako attachment

⚠️ Pořadí volání set_document() záleží: první dokument je tělo e-mailu, každý další je připojen jako příloha.

Alternativa: multipart/mixed přístup

" 1. Sestav HTML tělo
DATA(multipart) = NEW cl_gbt_multirelated_service( ).
multipart->set_main_html( EXPORTING content = body_html_soli ).

" 2. Přidej přílohu do multipart
DATA(lo_att_service) = NEW cl_gbt_part_service( ).
lo_att_service->set_content(
    i_content_type = 'application/pdf'
    i_content      = lt_solix
).
lo_att_service->set_content_disposition(
    i_disposition = 'attachment'
    i_filename    = 'report.pdf'
).
multipart->add_part( lo_att_service ).

" 3. Vytvoř dokument z multipart
DATA(doc_bcs) = cl_document_bcs=>create_from_multirelated(
    EXPORTING
        i_subject          = lv_subject
        i_multirel_service = multipart
).

11. Napojení na Output Management a BRF+

E-Mail Templates jsou primárně navrženy pro použití s Output Managementem (Nemo Output Management) v S/4HANA. Lze je ale volat i zcela samostatně.

Architektura Output Managementu

Obchodní doklad (FV, FI, SD...)
       ↓
Output Condition (BRF+)
       ↓
Output Type → přiřazena E-Mail Template
       ↓
Automatické odeslání e-mailu

Konfigurace v BRF+

  1. Otevři BRF+ Workbench (transakce BRF+).
  2. Najdi příslušnou Output Condition aplikaci (např. FIN_CUSINV_EMAIL_TEMPL pro faktury).
  3. V Action části nastav:
    - Output Channel: EMAIL
    - E-Mail Template ID: ID tvé šablony
    - Language Determination: z dokladu nebo pevně

Standardní šablony (příklady)

Template ID Oblast použití
FIN_CUSINV_EMAIL_TEMPL Zákaznické faktury (FI)
SD_ORCO_EMAIL_TEMPL Potvrzení objednávky (SD)
MM_PORDERS_EMAIL_TEMPL Objednávky (MM)

Pro vlastní custom šablony vždy používej prefix Z.

14. Doporučená architektura pro hromadné odesílání

Při odesílání e-mailů většímu počtu příjemců (background job) doporučujeme tento pattern:

REPORT Z_SEND_EMAILS.

SELECTION-SCREEN:
  P_TEST  (testovací mode – nevysílá e-maily)
  P_RECP  (alternativní příjemce pro testování)

START-OF-SELECTION:
  1. Načti vstupní data (SELECT / CDS view)
  2. Agreguj data per příjemce (LOOP + DELETE DUPLICATES)
  3. Pro každého příjemce:
     a. Sestav dynamický HTML obsah (pokud je tabulka)
     b. Inicializuj CL_SMTG_EMAIL_API + CL_BCS
     c. render() → REPLACE placeholder
     d. Sestav multipart zprávu
     e. Nastav sender, recipient
     f. IF p_test = abap_false → bcs->send()
     g. COMMIT WORK AND WAIT
     h. Loguj výsledek (WRITE, ALV, appl.log)
  4. Naplánuj jako background job (SM36)

Doporučení pro background job

  • Používej systémového uživatele jako odesílatele (např. SAP_WFRT, ne dialog user).
  • Spouštěj job pouze v produkci, v DEV/QA používej testovací mode s P_RECP.
  • Loguj výsledky do Application Log (CL_BAL_APPLICATION_LOG) pro auditovatelnost.
  • Nastav správný Send Job v SCOT – bez něho zůstanou e-maily ve frontě SOST.

15. Troubleshooting

15.1 Šablona není vidět v SMTG_WB_START

# Příčina Řešení
1 Šablona není aktivována V SE80 označ objekt → Ctrl+F3 (Activate). Neaktivní objekty se v SMTG_WB_START nezobrazují.
2 Filtr na selection screenu Spusť SMTG_WB_START s prázdnými filtry → F8.
3 Šablona uložena v $TMP Local objects lze vidět pouze v systému kde byly vytvořeny. Přesuň do transportovatelného balíku.
4 Chybí oprávnění Zkontroluj auth. objekt S_DEVELOP (aktivita 16) nebo S_SMTG_API.
5 Objekt zamčený v transportu Zkontroluj v SE80 záložku Properties nebo přes SE03.

Rychlý test: Jdi do SE80 → v dropdown vyber Email Template → zadej název objektu ručně. Pokud ho SE80 najde a je Active, jde o problém s filtrem/oprávněním v SMTG_WB_START.

15.2 E-mail se neodesílá (zůstává ve frontě SOST)

# Příčina Řešení
1 Send Job v SCOT neběží SCOT → Jobs → Start Send Process nebo zkontroluj plánování jobu v SM37.
2 SMTP node není nakonfigurován V SCOT zkontroluj SMTP node, otestuj přes SOSG.
3 Příjemce nemá e-mailovou adresu Zkontroluj, zda cl_cam_address_bcs::create_internet_address() dostává neprázdnou hodnotu.
4 COMMIT WORK chybí Po bcs->send() musí následovat COMMIT WORK AND WAIT.
5 Sandbox/dev system blokuje odeslání Zkontroluj SCOT → Settings → General Settings – povolené domény odesílání.

15.3 CDS view nefunguje / placeholder se nenahrazuje

# Příčina Řešení
1 CDS view není aktivní Aktivuj CDS v SE80 nebo ADT.
2 Název pole neodpovídá Název v {{placeholder}} musí přesně odpovídat aliasu pole v CDS (case-insensitive, ale bez mezer).
3 CDS key nevrací data Zkontroluj, zda předávaná hodnota v it_data_key odpovídá existujícímu záznamu v DB.
4 Špatný název CDS ve šabloně V záložce Header šablony musí být správný technický název CDS view (ne sqlViewName).

15.4 Exception CX_SMTG_EMAIL_COMMON

Nejčastější příčiny:
- Template ID neexistuje nebo není aktivní.
- CDS view přiřazená šabloně neexistuje v systému.
- Jazyk zadaný v iv_language nemá v šabloně variantu a ani fallback EN neexistuje.

CATCH cx_smtg_email_common INTO DATA(lx).
    " Detailní chybová zpráva:
    WRITE / lx->get_text( ).
    " Nebo zaloguj do application logu pro background job
ENDTRY.

16. Příklad exception handlingu

TRY.
    " ... kód pro odeslání e-mailu ...

CATCH cx_smtg_email_common INTO DATA(lx_email).
    " Chyba šablony (neexistuje, neaktivní CDS, chybějící jazyk)
    WRITE / |CHYBA šablony: { lx_email->get_text( ) }|.

CATCH cx_send_req_bcs INTO DATA(lx_bcs).
    " Chyba BCS (odeslání, nastavení dokumentu)
    DATA(ls_msg) = VALUE #(
        msgid = sy-msgid msgno = sy-msgno
        attr1 = sy-msgv1 attr2 = sy-msgv2
        attr3 = sy-msgv3 attr4 = sy-msgv4
    ).
    WRITE / |CHYBA BCS: { ls_msg-msgid } { ls_msg-msgno }|.

CATCH cx_address_bcs INTO DATA(lx_addr).
    " Chyba adresy příjemce/odesílatele
    WRITE / |CHYBA adresy: { lx_addr->get_text( ) }|.

ENDTRY.

17. Tipy a doporučení

HTML šablona

  • ✅ HTML vždy připravuj a testuj v VS Code (Live Preview, Emmet) – teprve poté kopíruj do SE80.
  • ✅ Používej inline CSS – e-mailoví klienti (Outlook, Gmail) nepodporují stylesheets v <head>.
  • ✅ Testuj layout v různých e-mailových klientech (Outlook, Gmail, Thunderbird mají různé chování).
  • ⚠️ Vyhni se JavaScriptu v HTML – e-mailoví klienti ho blokují.
  • ⚠️ Obrázky z externích URL nemusí být zobrazeny (blokování obsahu) – preferuj inline Base64 nebo logo přímo v textu.

ABAP a výkon

  • ✅ Při hromadném odesílání vždy seskupuj záznamy per příjemce (jeden e-mail = jeden příjemce, více dat v těle).
  • ✅ Testuj vždy nejdříve s testovacím mode (parametr P_TEST, alternativní adresa P_RECP).
  • ✅ Po send() vždy volej COMMIT WORK AND WAIT.
  • ✅ Používej smysluplné pojmenování HTML placeholderů (<!--///EmpContent-->).
  • ⚠️ CDS view přiřazená šabloně musí existovat a být aktivní v každém systému (DEV/QA/PROD).
  • ⚠️ Dynamické proměnné {{NazevPole}} musí přesně odpovídat názvům/aliasům polí v CDS.

Odesílání a provoz

  • ✅ Pro odesílatele background jobů používej systémového uživatele (SAP_WFRT nebo vlastní batch user) – ne dialog usera.
  • ✅ Loguj výsledky odesílání do SOST nebo Application Logu pro auditovatelnost.
  • ✅ Monitoruj frontu SOST – zejména po prvním nasazení.
  • ✅ Před nasazením do PROD otestuj celý flow v QA s reálnými daty.

18. Checklist – rychlý přehled kroků

Příprava systému

  • Ověřit konfiguraci SCOT (SMTP node, Send Job)
  • Otestovat odesílání přes transakci SOSG

Vývoj šablony

  • Vytvořit CDS view se zdrojovými daty a klíčovými poli
  • Aktivovat CDS view v systému
  • Napsat HTML obsah v VS Code, otestovat layout
  • Vytvořit E-Mail Template v SE80 (nebo přes SMTG_WB_START)
  • Přiřadit CDS view v záložce Header šablony
  • Definovat CDS Fields (záložka CDS Fields)
  • Vložit HTML do záložky Body HTML
  • Přidat dynamické proměnné {{NazevPole}} do Subject/Body
  • Pro tabulkový obsah přidat HTML placeholder <!--///NazevPlaceholderu-->
  • Přidat vícejazyčné varianty (záložka Languages), vždy EN jako základ
  • Aktivovat šablonu (Ctrl+F3)
  • Otestovat Preview přímo v SE80

Vývoj ABAP programu

  • Napsat program s voláním CL_SMTG_EMAIL_API
  • Implementovat testovací mode (bez odesílání / alternativní adresa)
  • Implementovat exception handling (CX_SMTG_EMAIL_COMMON, CX_SEND_REQ_BCS, CX_ADDRESS_BCS)
  • Otestovat v DEV systému s testovacím mode
  • Otestovat v QA systému s reálnými daty
  • Naplánovat jako background job (SM36) pokud je potřeba

Provoz

  • Monitorovat SOST po prvním spuštění
  • Ověřit přijetí e-mailu a správné zobrazení v různých klientech
